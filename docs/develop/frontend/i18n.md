# 国际化

CrystalFramework 前端使用 `react-i18next` 实现国际化，支持按需动态加载语言文件。语言文件位于 `web/src/i18n/locales/`，文件名使用标准语言标签（如 `zh-CN.ts`、`en-US.ts`）。

## Key 结构

所有国际化 key 按功能划分为以下顶层命名空间：

| 命名空间 | 用途 | 示例 |
|---------|------|------|
| `pages` | 页面级别的文案 | `pages.userManager.title` |
| `components` | 组件级别的文案 | `components.dashboard.myJoinedTenants.title` |
| `entityNames` | 实体名称 | `entityNames.user` |
| `enums` | 枚举值翻译 | `enums.tenantMemberStatus.4` |
| `menu` | 侧边栏菜单 | `menu.admin.users` |
| `api` | API 通用消息 | `api.sessionExpired` |

## Key 命名规范

- 全部使用 **camelCase**
- 按层级从通用到具体排列：`{命名空间}.{模块}.{区域}.{具体字段}`
- **页面级别** key 以 `pages.{页面名}` 开头，适合页面标题、过滤条件标签、表单字段等
- **组件级别** key 以 `components.{组件名}` 开头，适合可复用组件的内部文案
- **列标题** 统一放在 `components.columns.{实体名}` 下
- 善用 **`{{variableName}}`** 插值替代字符串拼接，例如 `"共 {{total}} 条"`

## 如何添加

1. 在 `web/src/i18n/locales/{locale}.ts` 中添加翻译
2. 在组件中使用 `useTranslation()` 获取 `t` 函数

### 代码示例

```tsx
// locales/zh-CN.ts
pages: {
    myCustomPage: {
        title: '我的页面',
        subtitle: '这是我的自定义页面',
    },
    myCustomPage: {
        filter: {
            status: '状态',
            type: '类型',
        },
    },
},
components: {
    myComponent: {
        placeholder: '请输入...',
        confirmText: '确认',
    },
},

// 在页面组件中使用
import { useTranslation } from 'react-i18next';
import { ActionBarComponent } from '@/components/ActionBarComponent.tsx';

function MyPage() {
    const { t } = useTranslation();

    return (
        <ActionBarComponent
            title={t('pages.myCustomPage.title')}
            subtitle={t('pages.myCustomPage.subtitle')}
        />
    );
}
```

## 插值语法

语言文件中使用 `{{变量名}}` 作为占位符，`t()` 的第二个参数传入实际值：

```typescript
// 语言文件
{ "greeting": "你好，{{name}}！" }

// 组件中
t('greeting', { name: 'Alice' })
// 输出：你好，Alice！
```

## 枚举翻译

枚举值统一在 `enums` 命名空间下管理，必须严格执行以下四步流程，禁止在任何地方对枚举值使用硬编码标签。

### 第一步：定义前端枚举常量

后端定义的枚举必须在 `src/types/` 下有对应的 TypeScript 枚举。枚举值与后端字段保持一致：

```typescript
// src/types/tenant/tenant-benefit.types.ts
export enum TenantBenefitType {
    BOOLEAN = 0,
    LIMIT = 1,
    ENUM = 2,
}
```

::: warning 严禁魔法数字
所有组件和页面中必须使用枚举常量（如 `TenantBenefitType.BOOLEAN`）而非原始数字（`0`、`1`、`2`）。
:::

### 第二步：添加 i18n 翻译键

在 `src/i18n/locales/{locale}.ts` 的 `enums` 命名空间中添加翻译：

```typescript
// locales/zh-CN.ts
enums: {
    tenantBenefitType: {
        0: '开关',
        1: '数量限制',
        2: '枚举'
    },
}

// locales/en-US.ts
enums: {
    tenantBenefitType: {
        0: 'Boolean',
        1: 'Limit',
        2: 'Enum'
    },
}
```

### 第三步：注册获取函数

在 `src/i18n/enum-helpers.ts` 中注册获取函数，用于在组件中获取枚举值的翻译文本：

```typescript
// enum-helpers.ts
export function getTenantBenefitType(type: number): string {
    return translateEnum('tenantBenefitType', type);
}
```

### 第四步：在组件中使用

所有显示枚举值的地方必须通过获取函数获取翻译文本，禁止直接使用 `t('components.columns.xxx.typeBoolean')` 等分散的 i18n 键：

```tsx
import { TenantBenefitType } from "@/types/tenant/tenant-benefit.types.ts";
import { getTenantBenefitType } from "@/i18n/enum-helpers.ts";

// ✅ 正确：使用枚举常量 + 获取函数
<Select
    options={[
        { value: TenantBenefitType.BOOLEAN, label: getTenantBenefitType(TenantBenefitType.BOOLEAN) },
        { value: TenantBenefitType.LIMIT, label: getTenantBenefitType(TenantBenefitType.LIMIT) },
    ]}
/>

// ✅ 正确：表格列渲染
render: (_, row) => <Tag>{getTenantBenefitType(row.featureType)}</Tag>

// ❌ 错误：魔法数字 + 分散 i18n 键
render: (_, row) => {
    const map = { 0: t('...typeBoolean'), 1: t('...typeLimit') };
    return <Tag>{map[row.featureType]}</Tag>;
}
```

### 完整对照

| 步骤 | 文件 | 内容 |
|------|------|------|
| 1 | `src/types/{module}/{entity}.types.ts` | `export enum XxxType { ... }` |
| 2 | `src/i18n/locales/{locale}.ts` | `enums.xxxType: { 0: '...', 1: '...' }` |
| 3 | `src/i18n/enum-helpers.ts` | `export function getXxxType(...)` |
| 4 | 组件/页面中 | `getXxxType(XxxType.XXX)` |

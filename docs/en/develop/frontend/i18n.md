# Internationalization

The CrystalFramework frontend uses `react-i18next` for internationalization with on-demand dynamic loading of language files. Translation files are located at `web/src/i18n/locales/` and are named using standard language tags (e.g., `en-US.ts`, `zh-CN.ts`).

## Key Structure

All i18n keys are organized under the following top-level namespaces:

| Namespace | Usage | Example |
|-----------|-------|---------|
| `pages` | Page-level text | `pages.userManager.title` |
| `components` | Component-level text | `components.dashboard.myJoinedTenants.title` |
| `entityNames` | Entity names | `entityNames.user` |
| `enums` | Enum value translations | `enums.tenantMemberStatus.4` |
| `menu` | Sidebar menu items | `menu.admin.users` |
| `api` | Common API messages | `api.sessionExpired` |

## Key Naming Conventions

- Always use **camelCase**
- Order from general to specific: `{namespace}.{module}.{section}.{field}`
- **Page-level** keys start with `pages.{pageName}`, suitable for page titles, filter labels, form fields, etc.
- **Component-level** keys start with `components.{componentName}`, suitable for reusable component internal text
- **Column headers** are placed under `components.columns.{entityName}`
- Use **`{{variableName}}`** interpolation instead of string concatenation, e.g., `"Total {{total}} item(s)"`

## How to Add

1. Add translations in `web/src/i18n/locales/{locale}.ts`
2. Use `useTranslation()` in your component to get the `t` function

### Code Example

```tsx
// locales/en-US.ts
pages: {
    myCustomPage: {
        title: 'My Page',
        subtitle: 'This is my custom page',
    },
    myCustomPage: {
        filter: {
            status: 'Status',
            type: 'Type',
        },
    },
},
components: {
    myComponent: {
        placeholder: 'Please enter...',
        confirmText: 'Confirm',
    },
},

// In a page component
import { useTranslation } from 'react-i18next';
import { ActionBarComponent } from '@/components/ActionBarComponent.tsx';

function MyPage() {
    const { t } = useTranslation();

    return (
        <ActionBarComponent
            title={t('pages.myCustomPage.title')}
            subtitle={t('pages.myCustomPage.subtitle')}
        />
    );
}
```

## Interpolation Syntax

Use `{{variableName}}` as placeholders in language files, and pass actual values as the second argument to `t()`:

```typescript
// Language file
{ "greeting": "Hello, {{name}}!" }

// In component
t('greeting', { name: 'Alice' })
// Output: Hello, Alice!
```

## Enum Translations

Enum values are managed under the `enums` namespace and registered in `enum-helpers.ts` with accessor functions:

```typescript
// In i18n-rules.ts type definition
enums: {
    tenantMemberStatus: { 0: string; 1: string; 2: string; 3: string; 4: string };
}

// In enum-helpers.ts registration
export const getTenantMemberStatus = (status: number): string =>
    i18n.t(`enums.tenantMemberStatus.${status}`);
```
